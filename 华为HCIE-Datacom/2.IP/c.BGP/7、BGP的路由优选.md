### 前置预处理步骤
BGP 在启动路由优选前，会**先过滤并丢弃下一跳不可达的路由**，此类路由不会进入后续选路流程。

### 11条选路规则
1.优选**Preferred-Value**属性值最==大==的路由。        
	华为的私有属性 （对于值默认为0，越大越优先的） 本地有效的属性
	可通过 peer 入方向路由策略、本地路由始发时的路由策略设置，无入方向强制限制。

2.优选**Local_Preference**属性值最==大==的路由。
	仅在本 AS 内的 IBGP 对等体之间传递，不能传出 AS，用于控制 AS 内的流量出口方向，默认值为 100，值越大优先级越高。

3.优选**本地始发**的BGP路由
	本地始发的路由优先级：优选手动聚合\>自动聚合\>network\>import\>从对等体学到的。    
	补充特征：聚合路由下一跳为 127.0.0.1  >  network/import-route 发布的路由下一跳为 0.0.0.0 > 从对等体学到的路由下一跳为邻居更新源地址。

4.优选**AS_Path**属性值最==短==的路由。  
	核心约束：
	- 联盟内的子 AS 号（`AS_Confed_Sequence/Set`）不计入 AS_Path 长度；
	- 聚合路由的`AS_Set`类型 AS_Path，无论包含多少个 AS 号，长度均按 1 计算。

5.优选**Origin**属性最优的路由
	优先级从高到低为：`IGP（i）` > `EGP（e）` > `Incomplete（?）`。
	补充说明：
	network 发布的路由为 IGP 类型，EGP 协议学来的为 EGP 类型，import-route 引入的路由为 Incomplete 类型。

6.优选**MED**属性值最==小==的路由。  
	用于控制邻居 AS 的流量入口方向，默认值为 0，值越小优先级越高。
	核心生效前提：
	- 华为默认仅当两条路由的**左邻 AS（AS_PATH 第一个 AS 号）完全相同**时，才会比较 MED 属性；
	- MED 默认仅在两个相邻 AS 之间传递，不会传给第三个 AS；
	- 可通过`compare-different-as-med`命令强制比较不同邻 AS 路由的 MED 属性。

7.优选从**EBGP**对等体学来的路由（BGP联盟除外，联盟内的EBGP与IGBP优先级相同）
	优先级规则：普通 EBGP 路由 > IBGP 路由 = 联盟 EBGP 路由。
	关键说明：联盟内跨子 AS 的联盟 EBGP 路由，与 IBGP 路由同级，不会被本条规则优先选择。

8.优选到**Next_Hop**的IGP度量值最==小==的路由。
	即 IP 路由表中，到达 BGP 路由下一跳地址的 IGP（OSPF/IS-IS/ 静态路由等）cost 值越小，路由越优先。

9.优选**Cluster_List**最==短==的路由
	 仅 RR 场景生效，Cluster_List 是 RR 反射路由时添加的集群 ID 列表，用于集群间防环，长度越短优先级越高；非 RR 场景该属性为空，长度为 0，所有路由本条规则平级。

10.优选**Originator_ID**/ 发布者 Router-ID 最==小==的路由  
	RR 场景下，优先比较`Originator_ID`（路由在本 AS 内原始始发者的 Router-ID）；
	无 Originator_ID 时，比较路由发布者的 Router-ID，值越小优先级越高。

11.优选具有最==小==IP地址的对等体通告的路由。  
	比较本地与邻居建立 BGP 对等体时配置的`peer x.x.x.x`中的邻居 IP 地址（即邻居的更新源 IP），地址越小优先级越高。

### 注意：
==默认BGP路由只会将最优的一条加入到IP路由表中==
在BGP中执行多条路由进行负载分担  
```
[AR-bgp]maximum load-balancing ibgp 2 # 配置IBGP/联盟EBGP路由最大负载分担条数 
[AR-bgp]maximum load-balancing ebgp 2 # 配置普通EBGP路由最大负载分担条数
```
- 该命令仅影响 BGP 路由向本地 IP 路由表的注入，不影响 BGP 路由的传递；
- 即使配置了负载分担，BGP 向对等体发布路由时，仍只会从负载分担路由中选出一条最优路由发布，不会发布所有负载分担条目。
