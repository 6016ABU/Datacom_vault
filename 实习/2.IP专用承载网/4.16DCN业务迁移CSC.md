# 一、fake-as
```R
# peer fake-as（BGP视图）(IPv4)

<HUAWEI> system-view
[~HUAWEI] bgp 100
[*HUAWEI-bgp] peer 10.1.1.2 as-number 200
[*HUAWEI-bgp] peer 10.1.1.2 fake-as 99
```

**应用场景**

该命令常用于运营商修改网络部署的场景。例如在运营商之间的收购和合并场景中，由于收购运营商和被收购运营商位于不同的AS，这就需要把被收购运营商中的BGP邻居迁移到收购运营商的AS中。但在网络迁移的过程中，被收购运营商的客户可能不期望修改本地的BGP配置或者==不想立即修改==本地的BGP配置，可能会导致BGP邻居较长时间的中断。

如果配置**peer fake-as**命令而不指定其他参数，则建立BGP邻居时只能使用伪AS号，在发布路由时只增加fake-as到路由AS_Path列表中。

**前置条件**

先使用**peer as-number**命令成功创建对等体后，才可以对该对等体指定伪AS号。
配置**peer fake-as**命令会导致邻居重新建立，使已经建立的邻居断连，可能导致业务中断，请谨慎配置。

# 二、routing-table limit（VPN实例视图）

## 命令功能

**routing-table limit**命令用来限制当前VPN实例相应地址族下所能容纳的路由数。

**undo routing-table limit**命令用来恢复缺省配置。

缺省情况下，VPN实例相应地址族所能容纳的路由数没有限制，但同一设备上所有的私网路由和公网路由的总和不能超过设备可容纳的单播路由总数。

**应用场景**

为了防止VPN实例IPv4/IPv6地址族的路由表注入过多路由，可通过配置routing-table limit限制当前VPN实例相应地址族下所能容纳的路由数。当VPN实例IPv4/IPv6地址族的路由表中路由超过阈值，则不会再向邻居发布路由。 当执行**undo routing-table limit**命令后，系统将把由于超限未添加到VPN路由表的路由重新添加到本VPN路由表中。

```R
华为的配置：
ip vpn-instance wangguan
 ipv4-family
  undo routing-table limit
  vpn-target 24059:38070 export-extcommunity
  vpn-target 24059:38070 import-extcommunity

中兴的配置：
ip vrf ChinaMobile_CQ_BOSS
  address-family ipv4
    no maximum routes

```

# 三、ip urpf（接口视图）

**URPF (Unicast Reverse Path Forwarding，单播逆向路径转发)** 是华为 NE40E 路由器上的一种**防 IP 地址欺骗**的安全技术，用于校验报文源 IP 的合法性。
## 命令功能

**ip urpf**命令用来在相应的接口视图下使能URPF检查。

**undo ip urpf**命令用来在相应的接口视图下去使能URPF检查。

缺省情况下，URPF检查没有使能。
## 参数说明

| 参数                | 参数说明                                                                                                                                              | 取值  |
| :---------------- | :------------------------------------------------------------------------------------------------------------------------------------------------ | :-- |
| **strict**        | URPF将进行严格检查。即根据接收到的报文的源IP地址检查FIB表，找到单板号、端口号和VLAN ID（如果是VLAN报文），将它们和这个报文入口的单板号、端口号和报文中携带的VLAN ID （如果是VLAN报文）进行比较，如果相同则URPF检查通过，报文正常转发。如果不同，则丢弃此报文。 | -   |
| **allow-default** | 对匹配缺省路由的报文进行转发。                                                                                                                                   | -   |
| **loose**         | URPF将进行松散检查。即根据接收到的报文的源IP地址检查FIB（Forward Information Base）表，如果找到出接口则通过；如果找不到出接口，则丢弃。                                                              | -   |
| **statistics**    | URPF统计功能。                                                                                                                                         | -   |
| **enable**        | 使能URPF统计功能。                                                                                                                                       | -   |

```豆包
### 一、工作原理

当报文进入 NE40E 时，设备会：

1. 提取报文的**源 IP 地址**
2. 在**FIB 转发表**中查找该源 IP 的路由
3. 检查：**报文入接口**是否等于**路由出接口**
    
    - ✅ 匹配：合法，允许转发
    - ❌ 不匹配：判定为伪造 IP，**直接丢弃**
    

### 二、两种工作模式（NE40E 支持）

#### 1. 严格模式 (Strict)

- **条件**：源 IP 路由存在 **且** 入接口 = 路由出接口
- **适用**：**路由对称**、单路径、高安全场景（如边界、接入）
- **命令**：`ip urpf strict`

#### 2. 松散模式 (Loose)

- **条件**：**仅要求源 IP 路由存在**（不检查接口匹配）
- **适用**：**路由不对称**、多路径、复杂组网
- **命令**：`ip urpf loose`

#### 3. 允许缺省路由 (allow-default)

- 与`strict`配合：匹配**缺省路由**时也做接口校验
- 命令：`ip urpf strict allow-default`

### 三、NE40E 典型应用场景

- **运营商 / 企业边界**：防止公网伪造源 IP 攻击
- **数据中心**：防止内网 IP 欺骗、DDOS、SYN Flood
- **接入层**：防止用户伪造 IP 攻击核心

### 四、配置示例（NE40E）
# 接口下启用严格URPF
interface GigabitEthernet 1/0/0
 ip urpf strict
# 或启用严格+允许缺省路由
 ip urpf strict allow-default
# 或启用松散URPF
 ip urpf loose
 
 ### 五、核心作用
- **防 IP 欺骗**：阻断伪造源 IP 的恶意流量
- **抗 DDoS**：减少伪造源 IP 的攻击报文
- **提升安全性**：保护设备与内网

 
```

**应用场景**

一般情况下，设备接收到报文，获取报文的目的地址，针对目的地址查找路由。如果找到了就转发报文，否则丢弃该报文。URPF通过获取报文的源地址和入接口，以源地址为目的地址，在转发表中查找源地址对应的接口是否与入接口匹配，如果不匹配，认为源地址是伪装的，丢弃该报文。URPF通过反查源路由，如果命中黑洞路由，丢弃该报文。通过这几种方式，URPF就能有效地防范网络中通过修改源地址而进行的恶意攻击行为的发生。


```R
华为的配置：
interface GigabitEthernet 1/0/0
 ip urpf strict
中兴的配置：
interface gei-0/0/1/17.97
  ipv4 verify unicast source reachable-via any ignore-default-route
```
